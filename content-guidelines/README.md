## Key Principles Summary

### Brand
- We are **GTM Engineers**, not marketers
- We build **revenue infrastructure**, not campaigns
- We speak to **technical founders**, not generic SMBs

### Tone
- **Direct** – Get to the point
- **Technical** – Use precise language
- **Data-backed** – Support claims with evidence
- **Practical** – Focus on application, not theory

### SEO/AEO/GEO
- **SEO:** Keywords in titles, meta descriptions, headers
- **AEO:** Snippet-ready sections, FAQs, tables
- **GEO:** Original research, entity mentions, schema markup

### E-E-A-T
- **Experience:** Show real implementations and specific outcomes
- **Expertise:** Include technical depth and frameworks
- **Authority:** Publish original research, earn citations
- **Trust:** Be transparent, cite sources, update content

## How to Use These Guidelines

1. **Before writing:** Review Brand and Tone guides
2. **While writing:** Follow appropriate template
3. **Before publishing:** Run through SEO/AEO/GEO and E-E-A-T checklists
4. **Reference frequently:** Keep this folder bookmarked
